Historiquement comment est-on arrivé au Big Data

Comment en est on arrivé au Big Data ?

L'arrivée du Big Data est liée au progrès des systèmes de stockage et d'analyse de l'information numérisée

De plus l'explosion d'Internet dans les années 2000, le développement des réseaux sociaux et des technologies mobiles ont contribué à la naissance du Big Data

En effet depuis les années 2000 les échanges d'informations peut se faire n'importe où, n'importe quand ce qui a causé l'émergence de données trop volumineuse pour être manipulé avec des bases de données relationnelles, c'est ce qui caractérise le Big Data

Définition

Terme BigData signifie litéralement....Parmis les definitions celle de Wikepedia qualifie big data d'expression anglophone utilisée pour désigner des ensembles de données tellement gros qu’ils en deviennent difficiles à travailler avec des outils classiques de gestion de base de données. »
Les données en questions sont des données numérique et varient du message envoyé (SMS, IM, email), jusqu’à des transactions en ligne passant par les vidéos qu’on publie sur le net, nos positions géographique (géolocalisation), les résultats de recherches scientifiques et pleins d’autres sorte de données.

Big Data = Mégadonnées, grosses données ou données massives

Types de données

  • Messages que nous nous envoyons
  • Vidéos que nous publions
  • Informations climatiques
  • Signaux GPS
  • Enregistrements transactionneles d'achats en ligne
  • Et bien encore
Le Big Data est caractérisé par les 3V.
Volume : Quantité de données générée est en pleine expansion et suit une loi quasi exponentielle : e-commerce, réseaux sociaux. Vélocité (Rapidité) : rapidité renouvellement des données n'est plus à démontrer. Nouvelles mise à jours sont sotckés en respect de la devise du big data qui est de tout garder.
Variété : Bases de données spécialisées sont en mesure de gérer la multiplicité des formats de données : numérique, texte, image...

Les 3 V

Certaines personne du monde du Big Data pense qu’il n’est pas inutile d’ajouter un quatrième V pour la valeur des données, c’est-à-dire l’utilité de la donnée.

Un quatrième V ?

Certains caractérisent le Big Data avec un quatrième V qui est la Valeur

Domaines d'application

Le marketing

  • Les entreprises veulent comprendre les comportements et les attentes de leurs clients afin de mieux cibler leurs propositions
  • Elles créent des modèles prédictifs pour anticiper le départ d’un client ou les ventes d’un produit
  • Par exemple, les sites utilisaient jusqu'ici les cookies mais les plus grands sites utilisent le Big Data pour récupérer et analyser les données de navigation en provenance de sites tiers

La Santé

Avec la médecine prédictive, c’est-à-dire, prévoir un risque de maladie chez certains patients à partir du génome

Le Sport

Certains club posent sur les vêtements des joueurs des capteurs permettant de recueillir des informations sur les joueurs (transpiration, rythme cardiaque et respiratoire…) et ainsi prévenir les risques de blessures

Outils informatiques et outils mathématiques pour le Big Data

Les outils informatiques

Le cloud computing est l’exploitation de la puissance ou de stockage de serveurs informatiques distants par l’intermédiaire d’un réseau, généralement internet. Le cloud computing et le big data sont deux concepts liés car c’est grâce au cloud computing que nous pouvons traiter du Big Data sur nos laptop.

Cloud Computing = L'exploitation de la puissance de stockage de serveurs informatiques distants


Big Data et Cloud Computing, deux concepts liés

ll existes des milliers d’outils informatique pour le Big Data. Nous allons donc catégoriser ces différents outils en 7 parties et donner quelques exemples.

Big Data

  1. Gestion et Stockage de données (Data Storage)

  2. Nettoyage des données (Data Cleaning)

  3. Exploration de données (Data Mining)

  4. Analyse de données (Data Analysis)

  5. Visualisation de données (Data Visualisation)

Gestion et Stockage de données (Data Storage)

Hadoop qui est la solution Big Data la plus connue et la plus courante dans ce domaine. Il y a également Cloudera, MangoDB, Talend etc.

Nettoyage des données (Data Cleaning)

Avant que vous ne puissiez vraiment explorer des données selon leur utilités il faut les nettoyer. Quand on récupère des données nous n’avons pas systématiquement un ensemble de données propre, bien structuré. OpenRefine est un outil open source qui est dédié à nettoyer un ensemble de données désordonné, tout comme DataCleaner.
Le data mining a pour objet l’extraction d’un savoir ou d’une connaissance à partir de grandes quantités de données, par des méthodes automatiques ou semi-automatiques. Elle se propose d’utiliser un ensemble d’algorithmes issus de disciplines scientifiques diverses telles que les statistiques, l’intelligence artificielle ou l’informatique. Pour cela il existe RapidMiner, IBM SPSS Modeler, Oracle Data Mining, Teradata et plusieurs autres.

Analyse de données (Data Analysis)

Alors que l’exploration de données permet de parcourir toutes vos données à la recherche de modèles reconnus au paravant, l’analyse de données permet d’évaluer l’impact au fil du temps. L’analyse nous permet de poser des questions bien spécifique et de trouver leurs réponses dans les données. Voici une liste d’outils qui fait cela : Qubole, BigML, Statwing


Visualisation de données (Data Visualisation)

Les outils suivants permettent à vos données de prendre vies. En effet, pas tout le monde n’arrive à lire une base de données. C’est pour cela que Tableau, Silk, Chartio le permettent.

Les outils mathématiques

Géométrie

Trouver le bon espace ou un modèle de déformations

Statistique

  • Estimer la distance géodésique
  • Étudier la variabilité, plans d’expérience dynamiques, numériques

Théorie des graphes

Estimation de la distance en trouvant les "chemins les plus courts" entre les points

Optimisation

Calculer la distance minimale

Clustering (ou partitionnement de données)

Trouver des groupes similaires au sens de la distance

Apprentissage

Prévoir l’appartenance à un groupe, apprentissage renforcé

Probabilités

Analyse de graphes, matrices aléatoires, chaînes de Markov…
